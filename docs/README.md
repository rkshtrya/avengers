# Docs Folder Guide

This folder explains the project, the sources, the review plan, and the graph plan.

## Best Reading Order

| Step | File | Why |
| --- | --- | --- |
| 1 | `dataset-catalog.md` | Plain-English guide to every dataset: geography, source, type, and status. |
| 2 | `knowledge-graph-plan.md` | How the datasets become graph nodes, edges, and sourced claims. |
| 3 | `project-lattice-data-review.md` | Main review notes: what is available, what is missing, and what to do next. |
| 4 | `project-lattice-data-sources.md` | Longer product/data-source thinking from the project deck. |
| 5 | `team-collaboration-plan.md` | How to split review work across the team. |

## What To Look For

The main questions are:

- Can this data support a property knowledge graph?
- Can each user-facing claim point back to a source?
- What joins are easy?
- What joins still need GIS work?
- What data is missing before valuation can be trusted?

## Current Answer

The data is enough for a sourced explanation and knowledge-graph prototype.

Comparable sales are still the key missing data for a real valuation model.
